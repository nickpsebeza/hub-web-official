# hub-web-official

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存放，不针对特定域名或网站，主要服务于前端页面存档、示例展示或轻量级内容分发。

## 项目简介

这是一个通用的 HTML 页面归档仓库。仓库中存放的页面彼此独立，可能用于演示、测试、记录或作为小型功能页面的备份。仓库本身不绑定任何具体业务或域名，仅作为页面文件的存储与访问入口。

## 目录说明

```
.
├── index.html          # 仓库根页面（可自定义）
├── pages/              # 存放主要 HTML 页面文件
│   ├── page1.html
│   └── page2.html
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本说明文件
```

- `pages/`：核心目录，每个 HTML 文件即为一个独立页面。
- `assets/`：存放页面共享或专用的静态资源。

## 页面归档说明

- 每个 HTML 页面应保持完整独立，避免依赖外部不可控资源。
- 页面之间不应有强制链接依赖，但可通过相对路径引用 `assets/` 中的资源。
- 页面命名建议使用英文小写、短横线连接，例如 `about-page.html`。
- 如需添加新页面，直接在 `pages/` 下创建 HTML 文件，并在需要时更新根索引。

## 维护说明

- 本仓库不接收与具体商业推广、域名绑定或营销相关的内容。
- 提交新页面时，请确保文件内容合规、不违反相关法律法规。
- 建议保持目录结构清晰，及时清理不再使用的页面或资源。
- 如需修改仓库介绍或结构，请先通过 Issue 讨论。

## 使用方式

1. 克隆或下载本仓库。
2. 在本地通过浏览器直接打开 `pages/` 下的 HTML 文件，或部署到任意静态文件服务器。
3. 也可将仓库整体部署到支持静态页面的托管服务，直接访问各页面路径。
